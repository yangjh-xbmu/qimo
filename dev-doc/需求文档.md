# 戏剧与影视专业硕士个人网站项目需求文档

## 文档版本

- 版本号：1.0
- 创建日期：2024年1月
- 最后更新：2024年1月
- 项目状态：需求确认阶段

## 一、项目概述

### 1.1 项目背景

本项目旨在为戏剧与影视专业硕士林悦然创建一个个人展示与互动平台，将传统个人简历与AI对话功能相结合，展示个人专业成就、创作理念，并提供与访客的智能互动体验。

### 1.2 项目目标

- 建立现代化的个人品牌展示平台
- 提供专业的作品集展示功能
- 集成AI对话助手，限定讨论个人及专业相关话题
- 确保优秀的移动端用户体验
- 部署到Cloudflare Pages平台

### 1.3 目标用户

- 潜在雇主与合作伙伴
- 学术同行与研究人员
- 艺术机构与策展人
- 对戏剧与影视感兴趣的学生

## 二、功能需求

### 2.1 个人展示模块

#### 2.1.1 个人信息展示区

- 显示姓名、专业头衔、联系方式
- 展示个人头像（400x400px最佳）
- 简短的个人简介（100-150字）
- 地理位置信息

#### 2.1.2 作品集展示

- 网格化展示个人作品（最少4项）
- 每项作品包含：名称、类型、年份、描述、图片/链接
- 响应式网格布局（移动端：单列，桌面端：多列）
- 点击作品可查看详细信息或跳转到外部链接

#### 2.1.3 经历时间线

- 垂直时间线展示教育和工作经历
- 每个时间节点包含：时间、事件标题、详细描述、相关机构
- 按时间倒序排列（最新经历置顶）
- 支持不同类型经历的视觉区分（教育、工作、项目）

#### 2.1.4 价值观阐述

- 分段展示核心艺术理念、创作哲学、专业追求
- 采用卡片式设计，突出引述效果
- 适度的留白和排版强调重点内容

#### 2.1.5 学者展示区

- 网格展示欣赏的学者信息
- 每项包含：学者照片、姓名、研究领域、影响描述、推荐作品
- 支持学者信息的筛选和查看

### 2.2 AI对话功能模块

#### 2.2.1 对话界面

- 独立的聊天窗口组件
- 消息气泡式对话展示
- 清晰的输入区域和发送按钮
- 实时消息更新显示

#### 2.2.2 话题限制功能

- 系统级话题过滤：只允许讨论与林悦然个人及专业相关的话题
- 前端验证：用户输入时实时检查话题相关性
- 后端验证：API调用前的二次验证
- 明确的拒绝回答提示和话题引导

#### 2.2.3 对话管理

- 单次会话模式（不保存历史记录）
- 用户可随时开始新对话
- 对话状态指示（输入中、发送中、接收中）
- 错误处理和网络异常提示

#### 2.2.4 API集成

- Cloudflare Function作为代理层
- DeepSeek API后端集成
- 系统提示词精确控制回答范围
- API密钥的安全管理和环境变量配置

### 2.3 网站基础功能

#### 2.3.1 导航系统

- 固定顶部导航栏
- 响应式导航菜单（移动端汉堡菜单）
- 页面内锚点跳转
- 返回顶部功能

#### 2.3.2 页面布局

- 单页应用布局设计
- 清晰的页面分区
- 平滑的滚动过渡
- 一致的组件风格

#### 2.3.3 响应式设计

- 移动端优先的设计原则
- 四个主要断点：手机、平板、笔记本、桌面
- 触摸友好的界面元素
- 可读性优化的字体大小和间距

## 三、非功能需求

### 3.1 性能要求

- 页面首次加载时间小于3秒
- 对话响应时间小于5秒（依赖API响应）
- 移动端性能优化
- 图片懒加载和优化

### 3.2 可用性要求

- 符合WCAG 2.1 AA标准的无障碍访问
- 清晰的视觉层次和信息架构
- 直观的操作流程
- 全面的错误提示和帮助信息

### 3.3 安全要求

- 敏感信息（API密钥）不暴露在前端
- HTTPS加密传输
- 输入内容的安全过滤
- 跨域请求的安全限制

### 3.4 兼容性要求

- 支持现代浏览器（Chrome、Firefox、Safari、Edge）
- 移动端浏览器兼容
- 响应式设计适配不同设备尺寸

### 3.5 可维护性要求

- 清晰的代码结构和注释
- 模块化的组件设计
- 易于更新的内容管理系统
- 完善的文档说明

## 四、设计规范

### 4.1 视觉设计

#### 4.1.1 色彩方案

- 主色调：藏青色系（#102a43）
- 辅助色：蓝色系（#60a5fa）
- 背景色：白色和浅灰色
- 文字色：深灰色和黑色
- 强调色：蓝色系亮色

#### 4.1.2 字体系统

- 主要字体：Inter（英文字体）
- 中文字体：PingFang SC、Microsoft YaHei
- 标题字体：Playfair Display（可选）
- 字号层次：6级标题系统
- 行高和字间距优化

#### 4.1.3 间距系统

- 基础间距单位：8px
- 标准间距：16px、24px、32px、48px、64px
- 内容区域最大宽度：1200px
- 移动端边距：16px

#### 4.1.4 图标系统

- 统一的图标风格
- 适当的使用语义化图标
- 确保图标的可访问性
- 支持深色和浅色模式

### 4.2 交互设计

#### 4.2.1 动画效果

- 页面加载时的渐入效果
- 滚动触发的滑动效果
- 悬停状态的微妙反馈
- 过渡动画时长：200-300ms

#### 4.2.2 反馈机制

- 按钮点击的视觉反馈
- 表单验证的即时反馈
- 加载状态的明确指示
- 成功/失败操作的提示

#### 4.2.3 移动端交互

- 触摸目标的合适大小（最小44x44px）
- 手势操作的合理运用
- 键盘输入的良好支持
- 视口缩放的适当控制

### 4.3 内容策略

#### 4.3.1 内容结构

- 信息架构的清晰层级
- 内容分组的逻辑性
- 重点内容的突出展示
- 辅助信息的适当隐藏

#### 4.3.2 文案风格

- 专业但不失亲切的语气
- 简洁明了的表达
- 一致的术语使用
- 适当的专业术语解释

## 五、技术需求

> 具体技术实现方案请参考 [技术方案.md](./技术方案.md)

### 5.1 技术栈要求

- 前端框架：React 18+ 配合 TypeScript
- 样式方案：Tailwind CSS
- 构建工具：Vite或Create React App
- 部署平台：Cloudflare Pages
- API代理：Cloudflare Functions

### 5.2 数据管理

- 静态内容使用JSON文件管理
- 环境变量管理API密钥
- 本地存储用于临时数据
- 无数据库依赖

### 5.3 API集成

- DeepSeek API集成（通过代理）
- 自定义系统提示词
- 请求频率限制
- 错误处理机制

### 5.4 开发要求

- 版本控制：Git
- 代码质量：ESLint和Prettier
- 组件化开发
- 模块化CSS

## 七、项目约束

### 7.1 时间约束

> 详细开发计划请参考 [开发计划.md](./开发计划.md)

- 总开发时间：5个工作日
- 部署时间：1个工作日
- 测试时间：包含在开发时间内

### 7.2 资源约束

- 开发人员：1名全栈开发者
- 设计资源：现有设计规范，无需额外设计
- 预算限制：仅限于Cloudflare Pages免费套餐

### 7.3 技术约束

- 必须使用Cloudflare平台
- 必须集成DeepSeek API
- 必须支持移动端优先
- 必须采用无服务器架构

## 八、验收标准

### 8.1 功能验收标准

- 所有展示模块正常工作
- 对话功能准确响应
- 话题限制有效执行
- 移动端适配完美

### 8.2 性能验收标准

- PageSpeed Insights评分大于90
- 移动端加载时间小于3秒
- 对话响应时间可接受
- 内存使用合理

### 8.3 设计验收标准

- 视觉设计与规范一致
- 交互体验流畅自然
- 无障碍访问合规
- 跨浏览器兼容

## 九、项目风险与缓解

### 9.1 技术风险

- API服务不稳定：实现优雅降级和重试机制
- 移动端兼容性问题：充分测试和渐进增强
- 性能问题：优化策略和监控

### 9.2 内容风险

- 个人隐私泄露：严格控制展示信息
- 版权问题：确保内容合法性
- 数据准确性：定期更新和维护

### 9.3 运营风险

- API费用超支：设置使用限制和监控
- 恶意使用：实现频率限制和验证
- 内容过时：建立更新机制
